# Техническое задание: for_meets

## Назначение

Сервис для захвата аудио с совещаний и последующей генерации структурированных документов.
Личное использование. Основной язык встреч — русский.

---

## Источники аудио

Все источники обрабатываются единым способом — **захват системного аудио** (WASAPI loopback на Windows, BlackHole/CoreAudio на Mac). Пользователь открывает источник в любом плеере или браузере, приложение пишет то, что воспроизводится.

Поддерживаемые сценарии:
- Локальное видео (любой плеер)
- Текущее совещание в Google Meet
- Запись в приватном облаке (воспроизведение через браузер)
- Запись на онлайн-платформе (воспроизведение через браузер)

Дополнительно: пользователь может выставить максимальную скорость воспроизведения — приложение продолжает корректно захватывать аудио. Возможность "замутить" источник: звук захватывается, но не воспроизводится в наушниках.

## Режим работы

### Фаза 1 — Live (во время встречи)

**Автоопределение Google Meet:**
- Браузерное расширение отслеживает открытие вкладки с `meet.google.com`
- Автоматический старт захвата + системное уведомление "Запись началась" с кнопкой **"Эту встречу не записывать"**
- При закрытии вкладки — остановка захвата и переход к фазе 2

**Захват аудио:**
- Запись системного звука чанками, сохранение на диск
- Быстрая оценка качества каждого чанка (confidence score)
- При низком качестве чанка — ненавязчивый toast-попап: _"Плохо слышно последние ~30 сек"_
  - Без перехвата фокуса
  - Пользователь может самостоятельно переспросить собеседника

**Ручной запуск:**
- Через tray-меню открывается диалоговое окно выбора источника
- В диалоге отображаются:
  - Системные аудиоустройства
  - Открытые вкладки браузера с их названиями (через расширение)
- В этом же диалоге задаётся **название совещания** и **агенда** (см. ниже)
- Захват стартует сразу после подтверждения — до начала воспроизведения
- Режим ожидания сигнала: тишина в начале не считается ошибкой, оценка качества начинается только после появления звука

**При автозапуске (Google Meet):**
- Всплывает упрощённый диалог с полями: название совещания и агенда
- Можно заполнить или пропустить — запись уже идёт

**Браузерное расширение:**
- Используется для: списка всех открытых вкладок + автодетекта Google Meet
- Первоначальная настройка через скрипт `setup` (см. раздел "Первоначальная настройка")
- Если расширение не обнаружено — ссылка на установку прямо в диалоге

### Фаза 2 — Post-processing (по запросу)

Запускается:
- Попап сразу после окончания встречи: **"Обработать сейчас?"**
- Через tray-меню → раздел **"Необработанные задания"**

Три последовательных прохода, результат — три отдельных `.md` файла:

| # | Проход | Результат |
|---|--------|-----------|
| 1 | Транскрипция аудио | Подробная расшифровка встречи |
| 2 | Смысловой анализ (LLM) | Ключевые тезисы, решения, договорённости |
| 3 | Генерация follow-up (LLM) | Структурированный фоллоу-ап с задачами и следующими шагами |

## Выходные документы

### Именование файлов

Все файлы начинаются с названия совещания:
```
YYYY-MM-DD_[название совещания]_transcription.md
YYYY-MM-DD_[название совещания]_analysis.md
YYYY-MM-DD_[название совещания]_followup.md
```
Название совещания задаётся при старте записи (вручную или через диалог автозапуска). Если не задано — используется дата и время.

### Документ 1 — Расшифровка (`_transcription.md`)

Структура документа:
- **Метаданные в шапке:** название совещания, дата, длительность, участники
- **Агенда** (если была задана при старте)
- **Транскрипция с временными метками** от начала звукового потока:
  ```
  [00:01:23] Спикер 1: текст...
  [00:02:45] Спикер 2: текст...
  ```
- **Диаризация** — разделение по участникам (Whisper + pyannote.audio)
- **Идентификация спикеров:** если в ходе встречи участник называет своё имя или его называют — это имя фиксируется и используется вместо `Спикер N` для всей записи и в последующих расшифровках (сохраняется в базе сопоставлений)

### Документ 2 — Смысловой анализ (`_analysis.md`)

- Каждый тезис сопровождается:
  - Временной меткой (или несколькими, если тезис — синтез из разных моментов)
  - Автором (спикером)
- Формат:
  ```
  **Тезис:** текст мысли
  **Автор:** Имя / Спикер N
  **Источник:** [00:05:12], [00:18:34]
  ```
- Разделы: ключевые мысли, решения, договорённости, открытые вопросы

### Документ 3 — Follow-up (`_followup.md`)

- Структурированный список задач с ответственными и сроками (если упоминались)
- Следующие шаги
- Вопросы, требующие уточнения

## Интерфейс

**Tray-иконка (Windows) / Menu bar (Mac)** — приложение работает в фоне, не требует открытого окна.

Меню трея:
- Статус (записывает / ожидает)
- **Необработанные задания** — список завершённых встреч без документов
- **Обработанные задания** — архив
- Запустить запись вручную / остановить
- Настройки

---

## Первоначальная настройка

Скрипт `setup` при первом запуске:
1. Проверяет наличие зависимостей (Whisper, pyannote, ffmpeg и др.)
2. Устанавливает браузерное расширение (ссылка + инструкция)
3. Проверяет доступность системного аудиоустройства (WASAPI / BlackHole)
4. Проводит тестовую запись и транскрипцию
5. Выводит результат проверки — всё ли работает корректно

## Технологии

| Компонент | Решение |
|-----------|---------|
| Захват аудио (Windows) | WASAPI loopback |
| Захват аудио (Mac) | BlackHole / CoreAudio |
| Транскрипция | Whisper |
| Диаризация (разделение спикеров) | pyannote.audio |
| Смысловой анализ и follow-up | Claude API (Anthropic) |
| Автодетект и список вкладок | Браузерное расширение |
| Формат выходных документов | Markdown (`.md`) |
| Язык встреч | Русский |
