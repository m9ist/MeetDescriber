## UX / UI

При RMS < `SILENCE_THRESHOLD_RMS` `_smoothed` сбрасывается в ноль без затухания.

**B2 — Попап "Начать запись": кнопка "Остановить сейчас"**
В уведомлении о старте записи (появляется при auto-detect Google Meet) добавить

**B4 — Не показывать "нечёткая речь" после остановки записи** ✅
Тост quality_warning полностью убран (#20, коммит `7fa3ffe`). Спектр-визуализатор в окне записи даёт достаточную обратную связь о качестве захвата. Quality-assessment в `audio_capture._evaluate_quality` остался, но без UI-нотификации.

**B5 — Трей: дата задания с часами и минутами** ✅

**B15 — Разделить анализ и follow-up в контекстном меню** ✅
Статус `analyzed` между `transcribed` и `done`. Tray показывает «→ анализ» / «→ follow-up»
и отдельные кнопки «Запустить анализ» / «Запустить follow-up» в зависимости от этапа.
`_job_label` теперь показывает `YYYY-MM-DD HH:MM  Название`.

**B9 — Диалог названия/агенды при автозапуске Google Meet**
Сейчас при автодетекте Meet запись стартует сразу, название берётся из заголовка вкладки.
По ТЗ: показывать упрощённый диалог (название + агенда) поверх записи, чтобы пользователь
мог уточнить метаданные. Диалог не блокирует — запись уже идёт.

**B10 — Настройки в tray-меню**
Пункт "Настройки" предусмотрен в ТЗ, но не реализован.

**B11 — "Расширение не найдено" → ссылка на установку в диалоге**
Если Native Messaging хост не отвечает или расширение не установлено —

## Запись

**B6 — Механизм перезапуска записи**
Поддержка сценария запланированного длительного отсутствия: возможность
приостановить/продолжить запись без создания новой сессии, или авто-возобновление
после паузы (например, при возвращении звука после тишины N минут).

**B7 — Запрет сна компьютера при активной записи**
Не давать Windows уходить в сон/гибернацию пока идёт запись.
Реализация: `ctypes.windll.kernel32.SetThreadExecutionState(ES_CONTINUOUS | ES_SYSTEM_REQUIRED)`
при старте захвата, сброс при остановке.

**B12 — "Замутить" источник**
Звук захватывается и пишется, но не воспроизводится в наушники пользователя.
Актуально при записи чужого видео/совещания фоном.
Реализация: WASAPI loopback + mute системного устройства вывода или отдельный silent sink.

## Сделано вне исходного бэклога

**Двухмодельный пайплайн транскрипции** ✅
GigaAM v2_rnnt (Sber, default на Windows) рядом с faster-whisper. Переключатель `TRANSCRIPTION_ENGINE`. Исследование: `research/transcribe_compare/`.

**Subprocess-изоляция тяжёлых ML-стадий** ✅
ctranslate2 и PyTorch конфликтуют по `cudnn64_9.dll` → BEX64 crash. Решение — три worker-процесса (diarize / transcribe-whisper / transcribe-gigaam), JSON через stdout, `PROGRESS:` через stderr.

**Окно «Все совещания»** ✅
Список всех сессий с фильтрами, контекстное меню, ручной запуск отдельных этапов. Колонка «Аудио» показывает session_id для физического доступа к папке `data/recordings/session_<id>/`.

**`tools/transcribe_file.py`** ✅
Standalone-CLI для разовой транскрипции одного файла (mp4/mkv/wav/...) без записи в БД. Опционально с диаризацией. См. `docs/TRANSCRIBE_FILE.md`.

**Alert drift mic↔loopback** ✅
В `_flush_chunk` мониторим расхождение mic-stream и loopback. При >5 сек — toast «⚠ Микрофон лагает» (другое приложение держит mic exclusive или USB-throttling).

**Логирование действий пользователя** ✅
`app.user_actions.log_action(name, **kwargs)` — все клики по tray/меню/диалогам пишутся в `app.log` с префиксом `app.user_action`.

**Issues #20, #21** ✅
Убрана тост-табличка «Плохо слышно» (#20). Полировка `MeetingStartDialog` (#21): автофокус, убран селектор источника, починен copy-paste на русской раскладке.
