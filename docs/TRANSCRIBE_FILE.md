# Транскрипция одного файла (без БД)

Утилита для прогона произвольного аудио/видео через тот же whisper+pyannote
pipeline, что использует основное приложение, — но без захода в БД и без
LLM-этапов (анализ, follow-up). Удобно, когда есть готовая запись (Zoom, Teams,
телефон), а основное приложение для неё не вели сессию.

## Использование

```bash
# С диаризацией (по умолчанию):
python tools/transcribe_file.py <путь_к_файлу>

# Без диаризации (только сплошной текст с таймстампами):
python tools/transcribe_file.py <путь_к_файлу> --no-diarize

# Свой путь вывода:
python tools/transcribe_file.py <путь_к_файлу> -o <путь_к_выводу.md>
```

По умолчанию результат пишется рядом с исходником:
`<имя_файла>_transcription.md`.

## Поддерживаемые форматы

faster-whisper использует **PyAV** для декодирования — работает прозрачно с
большинством контейнеров и кодеков:

- Видео: `.mp4`, `.mkv`, `.webm`, `.mov`, `.avi`
- Аудио: `.wav`, `.mp3`, `.m4a`, `.flac`, `.opus`, `.ogg`

`ffmpeg` в PATH **не** требуется (PyAV статически линкует свой FFmpeg).

## Что внутри

1. **Извлечение аудио** — через PyAV исходник декодируется в `audio_16k_mono.wav`
   (16 kHz mono) во временной директории. Файл нужен в этом формате для
   pyannote.
2. **Диаризация** (если не выключена) — subprocess `diarize_worker.py` грузит
   pyannote.audio на CUDA и размечает спикеров.
3. **Транскрипция** — subprocess `transcribe_worker.py` грузит whisper-large-v3
   на CUDA, кушает тот же WAV.
4. **Выравнивание** — сегменты транскрипции сшиваются со спикерами по
   максимальному перекрытию (`_assign_speakers` из `app/processing/pipeline.py`).
5. **Имена** — попытка вытащить имена из текста ("меня зовут …", "я …"),
   иначе спикеры размечаются как «Спикер 1», «Спикер 2» и т.д.
6. **Markdown** пишется в output path.

Subprocess-изоляция обоих этапов нужна потому, что ctranslate2 и torch бандлят
разные версии `cudnn64_9.dll`, и при одновременной загрузке в одно адресное
пространство Windows крашит процесс через 8 минут (см. историю в
`docs/BACKLOG.md`).

## Требования

- Python 3.11 + установленные зависимости проекта (`pip install -r requirements.txt`)
- CUDA-видеокарта с ≥6 GB VRAM
- `HUGGINGFACE_TOKEN` в `.env` (для загрузки гейтед-моделей при первом запуске)

## Параметры whisper

Те же, что в `app/transcription/transcribe_worker.py`:

- `language=config.WHISPER_LANGUAGE` (ru по умолчанию)
- `word_timestamps=False` — не используется downstream, ускоряет
- `vad_filter=True` — отрезает тишину
- `temperature=(0.0,)` — без перебора температур (см. защита от зависаний)
- `condition_on_previous_text=False`, `repetition_penalty=1.2`,
  `no_repeat_ngram_size=3` — антициклинг для тихого аудио

## Пример вывода

```markdown
# Аудит процессов разработки - 2026_05_07 13_59 GMT+06_00

Длительность: 01:24:13  |  Язык: ru  |  Сегментов: 1299

---

**[00:00]** Всем привет, начнём через минуту.

**[00:18]** Окей, видно меня?

...
```

## Что **не** делает

- Не пишет в `data/meets.db` (как основное приложение)
- Не делает LLM-анализ или follow-up — только транскрипция + диаризация

Если нужны анализ/follow-up — кинь файл в основное приложение через
«Все совещания → Добавить вручную» (когда будет такой UI) или вручную создай
сессию в БД и положи WAV в `data/recordings/session_<id>/`.
