# Транскрипция одного файла (без БД, без диаризации)

Утилита для прогона произвольного аудио/видео через тот же whisper-pipeline, что
использует основное приложение, — но без захода в БД и без диаризации.
Удобно, когда есть готовая запись (Zoom, Teams, телефон), а основное приложение
для неё не вели сессию.

## Использование

```bash
python tools/transcribe_file.py <путь_к_файлу>
python tools/transcribe_file.py <путь_к_файлу> -o <путь_к_выводу.md>
```

По умолчанию результат пишется рядом с исходником:
`<имя_файла>_transcription.md`.

## Поддерживаемые форматы

faster-whisper использует **PyAV** для декодирования — работает прозрачно с
большинством контейнеров и кодеков:

- Видео: `.mp4`, `.mkv`, `.webm`, `.mov`, `.avi`
- Аудио: `.wav`, `.mp3`, `.m4a`, `.flac`, `.opus`, `.ogg`

`ffmpeg` в PATH **не** требуется (PyAV статически линкует свой FFmpeg).

## Что внутри

1. Создаёт subprocess `transcribe_worker.py` (изолирует ctranslate2 от
   torch/pyannote — без этого Windows крашит процесс через 8 минут, см.
   историю в `docs/BACKLOG.md`).
2. Воркер грузит whisper-large-v3 на CUDA (требуется ~6 GB VRAM).
3. Транскрипция с параметрами из `config.py` (`WHISPER_LANGUAGE`).
4. Прогресс печатается на stderr в формате `XX.X%  MM:SS / MM:SS`.
5. Markdown с таймстампами пишется в output path.

## Требования

- Python 3.11 + установленные зависимости проекта (`pip install -r requirements.txt`)
- CUDA-видеокарта с ≥6 GB VRAM
- `HUGGINGFACE_TOKEN` в `.env` (для загрузки гейтед-моделей при первом запуске)

## Параметры whisper

Те же, что в `app/transcription/transcribe_worker.py`:

- `language=config.WHISPER_LANGUAGE` (ru по умолчанию)
- `word_timestamps=False` — не используется downstream, ускоряет
- `vad_filter=True` — отрезает тишину
- `temperature=(0.0,)` — без перебора температур (см. защита от зависаний)
- `condition_on_previous_text=False`, `repetition_penalty=1.2`,
  `no_repeat_ngram_size=3` — антициклинг для тихого аудио

## Пример вывода

```markdown
# Аудит процессов разработки - 2026_05_07 13_59 GMT+06_00

Длительность: 01:24:13  |  Язык: ru  |  Сегментов: 1299

---

**[00:00]** Всем привет, начнём через минуту.

**[00:18]** Окей, видно меня?

...
```

## Что **не** делает

- Не пишет в `data/meets.db` (как основное приложение)
- Не выделяет спикеров (нет диаризации) — все реплики без атрибуции
- Не делает LLM-анализ или follow-up

Если нужны спикеры/анализ — кинь файл в основное приложение через
«Все совещания → Добавить вручную» (когда будет такой UI) или вручную создай
сессию в БД.
