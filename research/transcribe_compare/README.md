# Сравнение моделей транскрипции (RU)

Сравниваем 4 решения на двух референсных записях:

| Модель | Стек | Особенности |
|--------|------|-------------|
| **baseline (faster-whisper / large-v3)** | ctranslate2 + CUDA | то что в приложении сейчас, с нашими параметрами anti-loop |
| **WhisperX** | faster-whisper + (опц.) wav2vec2 align + (опц.) pyannote diarize | свой VAD-чанкинг |
| **GigaAM v2_rnnt** | Sber, native RU RNN-T | + silero VAD для длинных файлов |
| **Wav2Vec2-XLS-R-Russian** | jonatasgrosman/wav2vec2-large-xlsr-53-russian | CTC, без пунктуации |

## Референсы

| Session | Title | Длительность | Особенность |
|---------|-------|--------------|-------------|
| 63 | Secret | 82.5 мин | длинная встреча — нагрузочный тест |

## Результаты

### Session 64 — короткая запись с длинной паузой (31.5 s)

В этом тесте важна **полнота покрытия**. Диаризация (pyannote) нашла 4 региона речи: 0–1с, 1.2–2.5с, 16.6–29.4с, 29.6–31.1с (между 3 и 15с — тишина).

| Модель | Wall (load + run) | Сегментов | VRAM peak | Что нашла | Кач-во |
|--------|-------------------|-----------|-----------|-----------|--------|
| baseline | 7.5 s | 1 | ~6 GB | только начало (0–2.4с) | ✅ «Всем пока, теперь что?» |
| WhisperX | 13.9 + 1.1 s | 1 | 1.6 GB | только конец (16+с) | ✅ «Ну, в общем, тут точка.» |
| GigaAM v2_rnnt | 2.1 + 1.1 s | **3** | **0.6 GB** | **начало + середина + конец** | ⚠ Без пунктуации: «чем» / «пока теперь все» / «а потом этот диф уже можно то есть ты просто делаешь джин или там далее» |
| Wav2Vec2-XLSR | 13.4 + 0.5 s | 2 | 1.8 GB | начало + конец | ❌ «вчем пока теперьсделайь рукамн сначалтак ты тугив гигитерлд псяч» |

**GigaAM — единственная модель что захватила и начало, и середину, и конец**. Никакая из whisper-based моделей не справилась.

### Session 63 — длинная встреча (82.5 min)

В этом тесте важны **скорость**, **связность текста** и **общее покрытие**.

| Модель | Wall (load + run) | Сегментов | VRAM peak | Скорость vs realtime |
|--------|-------------------|-----------|-----------|---------------------|
| baseline | ~620 s | 1250 | ~6 GB | 8x |
| WhisperX | 13 + 160 s = 173 s | 199 | 0.3 GB* | **31x** |
| GigaAM v2_rnnt | 2 + 180 s = 182 s (vad 9s + transcribe 171s) | 744 | 0.66 GB | 28x |
| Wav2Vec2-XLSR | 3 + 56 s = 59 s | 276 | 1.8 GB | **88x** |

\* VRAM peak у WhisperX заниженный, потому что мы выгружаем whisper-модель ПЕРЕД alignment'ом для экономии памяти. Реальный пик ~3 GB во время инференса.

### Качество текста (subjective)

Все 4 модели правильно поняли разговор. Различия:

- **baseline (whisper-large-v3)**: полная пунктуация, заглавные, отлично читается.
- **WhisperX**: тот же whisper-large-v3 под капотом, но сегменты короче и точнее по таймстампам.
- **GigaAM**: без пунктуации, без заглавных букв. Слова в основном правильные.
- **Wav2Vec2**: без пунктуации, БОЛЬШИЕ ошибки распознавания (слова искажены).

## Выводы

### Для основного pipeline'а (длинные встречи)

1. **WhisperX** — самая сильная замена baseline'у. **3.6× быстрее** (173s vs 620s), та же модель large-v3, родной chunking без багa с длинными паузами. Минусы: тяжёлая установка (другой torch), pyannote 4.x API ломается во встроенной диаризации — нужен патч/обход. **Очевидный кандидат на следующий шаг.**

2. **GigaAM v2_rnnt** — конкурентоспособен по скорости (3.4× быстрее baseline'а) и качеству на русском, но **нет пунктуации/капитализации**. Это критично для нашего LLM-пайплайна (анализ/follow-up читаются хуже). Использовать можно только как fallback для проблемных фрагментов.

3. **Wav2Vec2-XLSR** — **сверхбыстрый** (10.5× быстрее baseline'а), VRAM 1.8 GB, но качество текста плохое. Хорош только как **детектор речи** для проблемных сегментов или real-time превью.

### Для коротких записей с паузами

GigaAM — единственное решение что **покрывает все регионы речи**. Whisper-семейство фундаментально нестабильно на коротких аудио с длинными внутренними паузами.

### Гибрид (рекомендация)

```
1. Главный проход:    WhisperX (точные сегменты, пунктуация, скорость)
2. Coverage-check:    pyannote diarize → check что все speech-регионы покрыты
3. Fallback:          GigaAM v2_rnnt на регионы где WhisperX выдал пустоту
```

Wav2Vec2 в продакшене не нужен — точность ниже остальных, профита по скорости в нашем кейсе нет.

```bash

.venv/Scripts/python.exe runner_wav2vec2.py <wav> results/sessionN
```

- Python 3.10

- Whisper model: large-v3 (одинаковый везде где применимо)
