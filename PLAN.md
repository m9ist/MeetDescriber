# План реализации: for_meets

## Финальный стек

| Компонент | Решение | Примечание |
|-----------|---------|------------|
| Язык | Python 3.11+ | |
| Tray-приложение | `pystray` | Кроссплатформенный tray/menubar |
| Захват аудио (Windows) | `PyAudioWPatch` | WASAPI loopback |
| Захват аудио (Mac) | `sounddevice` + BlackHole | Виртуальное аудиоустройство |
| Транскрипция (Windows) | `faster-whisper` + CUDA | GPU-ускорение через NVIDIA |
| Транскрипция (Mac) | `mlx-whisper` | Apple Silicon (MLX framework) |
| Диаризация | `pyannote.audio 3.1` | Требует HuggingFace token + принятие лицензии |
| LLM-анализ | `anthropic` SDK + Claude API | Смысловой анализ и follow-up |
| Браузерное расширение | Chrome Manifest V3 + Native Messaging | Список вкладок + автодетект Meet |
| База данных | SQLite (файл `data/meets.db`) | Сессии, спикеры, статусы |
| Конфиг | `.env` | Токены, пути, настройки |
| Формат документов | Markdown `.md` | |

## Проверка гипотез

Перед или в рамках Этапа 0 явно проверяем рискованные допущения.
Если гипотеза не подтверждается — фиксируем и ищем альтернативу до того, как строим на ней логику.

| # | Гипотеза | Риск | Как проверяем | Критерий успеха |
|---|----------|------|---------------|-----------------|
| H1 | WASAPI loopback захватывает системный звук на Windows | Аудиодрайвер может блокировать loopback | Тестовая запись 10 сек любого звука через `PyAudioWPatch` | Файл записан, звук слышен |
| H2 | BlackHole захватывает звук на Mac | Требует ручной настройки aggregate device | Тестовая запись через `sounddevice` + BlackHole | Файл записан, звук слышен |
| H3 | faster-whisper (CUDA) даёт приемлемое качество для русского live-аудио | Модель обучена не на live/шумном аудио | Транскрипция 1 мин русской речи с микрофона/системного звука | WER субъективно приемлем, нет грубых ошибок |
| H4 | mlx-whisper работает на Mac M4 Pro | Молодой проект, возможны баги | Та же транскрипция на Mac | Аналогично H3 |
| H5 | pyannote.audio разделяет спикеров в русской речи | Модель обучена преимущественно на английском | Диаризация 2-минутного разговора двух людей | Правильно делит реплики ≥ 80% случаев |
| H6 | Confidence score Whisper коррелирует с реальным качеством | Внутренний score может не отражать ошибки в русском | Сравнить score плохого и хорошего фрагмента вручную | Плохой звук → низкий score, чистый → высокий |
| H7 | Native Messaging между Chrome и Python работает стабильно | Сложный механизм, платформозависимый | Минимальный хост + расширение: ping-pong сообщение | Расширение получает ответ от Python хоста |
| H8 | Транскрипция при 2x скорости воспроизведения приемлема | Высокий темп речи снижает точность Whisper | Транскрипция того же фрагмента на 1x и 2x, сравнение | Качество на 2x субъективно приемлемо |

## Этапы реализации

### Этап 0 — Скелет и setup-скрипт
- [ ] Структура папок и пустые модули
- [ ] `.env` загрузка конфига
- [ ] SQLite схема: таблицы `sessions`, `speakers`, `jobs`
- [ ] `setup.py`: проверка зависимостей, установка расширения, последовательный запуск проверок гипотез H1–H8
- [ ] Результат setup — читаемый отчёт: что прошло, что нет, что делать дальше

### Этап 1 — Захват аудио `[H1, H2]`
- [ ] `PyAudioWPatch` / `sounddevice` — запись системного звука чанками
- [ ] Сохранение чанков на диск
- [ ] Режим ожидания сигнала (silence detection)
- [ ] Оценка качества чанка (confidence на лету через mini-whisper) `[H6]`
- [ ] Toast при низком качестве

### Этап 2 — Браузерное расширение `[H7]`
- [ ] Chrome расширение: мониторинг вкладок, детект `meet.google.com`
- [ ] Native Messaging хост на Python
- [ ] Диалог выбора источника с вкладками браузера

### Этап 3 — Tray и UI
- [ ] `pystray` tray-иконка
- [ ] Меню: статус, необработанные/обработанные задания, ручной запуск
- [ ] Диалог запуска: название совещания + агенда
- [ ] Уведомление при автодетекте Meet + кнопка "не записывать"
- [ ] Попап "Обработать сейчас?" по окончании

### Этап 4 — Транскрипция и диаризация `[H3, H4, H5, H6, H8]`
- [ ] `TranscriptionBackend` абстракция
- [ ] `faster-whisper` (Windows/CUDA) `[H3]`
- [ ] `mlx-whisper` (Mac) `[H4]`
- [ ] `pyannote.audio` диаризация `[H5]`
- [ ] Совмещение транскрипции с диаризацией по временны́м меткам
- [ ] Идентификация спикеров по именам + сохранение в БД
- [ ] Генерация `_transcription.md`

### Этап 5 — LLM-обработка
- [ ] Промпт смыслового анализа с временны́ми метками и авторами → `_analysis.md`
- [ ] Промпт follow-up → `_followup.md`

### Этап 6 — Полировка
- [ ] Обработка ошибок и переподключение
- [ ] Логирование
- [ ] Тесты на ключевые модули
