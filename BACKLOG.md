# Бэклог

## UX / UI

- ~~**Спектр: мгновенный стоп при тишине**~~  ✅
  При RMS < `SILENCE_THRESHOLD_RMS` `_smoothed` сбрасывается в ноль без затухания.

- **Попап "Начать запись": кнопка "Остановить сейчас"**
  В уведомлении о старте записи (появляется при auto-detect Google Meet) добавить
  кнопку "Остановить сейчас" — остановить захват немедленно без ожидания окончания встречи.

- ~~**Модальное окно статуса обработки**~~  ✅
  `ProcessingStatusWindow`: показывает текущий этап + spinner-анимацию,
  закрывается автоматически по завершении.

- **Не показывать "нечёткая речь" после остановки записи**
  Уведомления quality_warning приходят из ThreadPoolExecutor с задержкой — могут
  появиться уже после того, как запись остановлена. Подавлять их, если сессия завершена.

- ~~**Трей: дата задания с часами и минутами**~~  ✅
  `_job_label` теперь показывает `YYYY-MM-DD HH:MM  Название`.

## Запись

- **Механизм перезапуска записи**
  Поддержка сценария запланированного длительного отсутствия: возможность
  приостановить/продолжить запись без создания новой сессии, или авто-возобновление
  после паузы (например, при возвращении звука после тишины N минут).

- **Запрет сна компьютера при активной записи**
  Не давать Windows уходить в сон/гибернацию пока идёт запись.
  Вариант реализации: `ctypes.windll.kernel32.SetThreadExecutionState(ES_CONTINUOUS | ES_SYSTEM_REQUIRED)`
  при старте захвата, сброс при остановке.

## Другое

- **Запуск без описания + редактирование по вкладке Chrome**
  Фича: запустить запись без заполнения метаданных, а название и агенду подтянуть
  автоматически из заголовка вкладки Chrome, в которой идёт звук (через Native Messaging).
  Пользователь может отредактировать в процессе совещания.
