## UX / UI

**B1 — Спектр: мгновенный стоп при тишине** ✅
При RMS < `SILENCE_THRESHOLD_RMS` `_smoothed` сбрасывается в ноль без затухания.

**B2 — Попап "Начать запись": кнопка "Остановить сейчас"**
В уведомлении о старте записи (появляется при auto-detect Google Meet) добавить
кнопку "Остановить сейчас" — остановить захват немедленно без ожидания окончания встречи.

**B3 — Модальное окно статуса обработки** ✅
`ProcessingStatusWindow`: этап + spinner + прогресс-бар транскрипции (MM:SS / MM:SS),
закрывается автоматически по завершении.

**B4 — Не показывать "нечёткая речь" после остановки записи**
Уведомления quality_warning приходят из ThreadPoolExecutor с задержкой — могут
появиться уже после того, как запись остановлена. Подавлять их, если сессия завершена.

**B5 — Трей: дата задания с часами и минутами** ✅
`_job_label` теперь показывает `YYYY-MM-DD HH:MM  Название`.

**B9 — Диалог названия/агенды при автозапуске Google Meet**
Сейчас при автодетекте Meet запись стартует сразу, название берётся из заголовка вкладки.
По ТЗ: показывать упрощённый диалог (название + агенда) поверх записи, чтобы пользователь
мог уточнить метаданные. Диалог не блокирует — запись уже идёт.

**B10 — Настройки в tray-меню**
Пункт "Настройки" предусмотрен в ТЗ, но не реализован.
Минимальный набор: путь к claude CLI, модель Whisper, порог качества.

**B11 — "Расширение не найдено" → ссылка на установку в диалоге**
Если Native Messaging хост не отвечает или расширение не установлено —
показывать в диалоге ручного запуска ссылку/инструкцию по установке.

## Запись

**B6 — Механизм перезапуска записи**
Поддержка сценария запланированного длительного отсутствия: возможность
приостановить/продолжить запись без создания новой сессии, или авто-возобновление
после паузы (например, при возвращении звука после тишины N минут).

**B7 — Запрет сна компьютера при активной записи**
Не давать Windows уходить в сон/гибернацию пока идёт запись.
Реализация: `ctypes.windll.kernel32.SetThreadExecutionState(ES_CONTINUOUS | ES_SYSTEM_REQUIRED)`
при старте захвата, сброс при остановке.

**B12 — "Замутить" источник**
Звук захватывается и пишется, но не воспроизводится в наушники пользователя.
Актуально при записи чужого видео/совещания фоном.
Реализация: WASAPI loopback + mute системного устройства вывода или отдельный silent sink.

## Обработка / Надёжность

**B13 — Обработка ошибок pipeline**
Сейчас при падении claude CLI или недоступности модели задание зависает в статусе error
без возможности продолжить. Нужно:
- Показывать внятное сообщение в статус-окне
- Добавить кнопку "Повторить с этапа X" в tray или статус-окне
- Таймаут и retry для subprocess claude

**B14 — End-to-end тест с нуля**
Полный прогон: запуск → запись реального совещания → обработка → проверка всех 3 документов.
Проверить H8: качество транскрипции при воспроизведении записи на скорости 2×.
